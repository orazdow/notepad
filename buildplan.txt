Build plan for notepadclone
===========================

Goal
----

Make the extracted source build as a normal Android project under:

    com.clemm.notepad

with visible app name:

    Notepad


1. Add Gradle project files
---------------------------

Create:

    settings.gradle
    build.gradle
    app/build.gradle

The app module should use:

    namespace "com.clemm.notepad"
    applicationId "com.clemm.notepad"

Add AndroidX dependencies:

    androidx.appcompat:appcompat
    androidx.core:core


2. Replace JADX-renamed dependency classes
------------------------------------------

The source still imports decompiled dependency class names. These should be
changed to real public AndroidX/AppCompat APIs.

Replace:

    androidx.appcompat.app.ActivityC0142c
        -> androidx.appcompat.app.AppCompatActivity

    androidx.appcompat.app.DialogInterfaceC0141b.a
        -> androidx.appcompat.app.AlertDialog.Builder

    androidx.appcompat.app.AbstractC0140a
        -> androidx.appcompat.app.ActionBar

    androidx.core.app.C0266a
        -> androidx.core.app.ActivityCompat

    p000a.p013d.p014d.C0042a
        -> androidx.core.content.ContextCompat

Then update method calls to normal API names, for example:

    m828k()
        -> getSupportActionBar()

    abstractC0140aM828k.mo783d(false)
        -> actionBar.setDisplayHomeAsUpEnabled(false)

    C0266a.m1680a(...)
        -> ActivityCompat.requestPermissions(...)

    C0042a.m225a(...)
        -> ContextCompat.checkSelfPermission(...)

    DialogInterfaceC0141b.a methods
        -> AlertDialog.Builder methods such as setTitle, setMessage, setView,
           setPositiveButton, setNegativeButton, setAdapter, setSingleChoiceItems,
           setCancelable, and show.


3. Fix resource references
--------------------------

Check for references to resources that came from decompiled AppCompat internals.

Known issue:

    R.layout.select_dialog_singlechoice

This was not copied into the clean clone. Prefer:

    android.R.layout.select_dialog_singlechoice

or create a tiny app-owned layout if needed.

Current minimal themes use:

    Theme.AppCompat.Light
    Theme.AppCompat

Those require the AppCompat dependency.


4. Clean AndroidManifest.xml
----------------------------

Remove decompiled APK-only manifest attributes:

    android:compileSdkVersion
    android:compileSdkVersionCodename
    platformBuildVersionCode
    platformBuildVersionName

Move SDK configuration to Gradle:

    minSdk
    targetSdk

The launcher activity already has:

    android:exported="true"

Keep that. Apps targeting Android 12 or higher must explicitly declare
`android:exported` for components with intent filters, and launcher activities
should use `true`.


5. Choose conservative SDK path
-------------------------------

Conservative first build:

    compileSdk: current installed SDK
    minSdk: 23 or lower if desired
    targetSdk: 29

Reason: the current export feature still uses legacy external storage:

    Environment.getExternalStorageDirectory() + "/Notepad"
    WRITE_EXTERNAL_STORAGE

Targeting 29 gives behavior closest to the old app while the clone is first made
buildable and debuggable.

Modern follow-up:

    Replace legacy export with Storage Access Framework, then raise targetSdk.


6. Storage/export migration
---------------------------

Current code writes exported notes to:

    /sdcard/Notepad/<title>.txt

This relies on legacy shared external storage. Android 11 scoped storage limits
this pattern when targeting newer SDKs.

Recommended modern replacement:

    Intent ACTION_CREATE_DOCUMENT
    type text/plain
    suggested filename from note title
    write content through ContentResolver.openOutputStream(uri)

After that, remove:

    WRITE_EXTERNAL_STORAGE
    Environment.getExternalStorageDirectory()
    runtime storage permission request


7. Keep these old behaviors initially
-------------------------------------

To reduce risk, keep these while making the first build work:

    SQLite database name: NotepadDB
    table: notes(date TEXT, title TEXT, content TEXT, password TEXT)
    rowid as note ID
    date format: yyyy/MM/dd HH:mm
    package/application ID: com.clemm.notepad
    visible app name: Notepad
    Java source language
    XML layouts
    PreferenceActivity


8. Expected build blockers
--------------------------

Most likely first compiler errors:

    decompiled AndroidX class names
    decompiled AlertDialog.Builder method names
    missing R.layout.select_dialog_singlechoice
    manifest attributes rejected by modern Gradle/manifest merger
    legacy storage permission/export behavior on newer target SDKs


9. Recommended order
--------------------

1. Add Gradle files. Done.
2. Clean manifest for Gradle. Done.
3. Replace AppCompat/Core decompiled imports and method calls. Done.
4. Fix `select_dialog_singlechoice`. Done.
5. Build.
6. Fix compiler errors from any remaining API or resource issues.
7. Run on device with `applicationId "com.clemm.notepad"`.
8. Test list, create, edit, delete, sort, lock/unlock, share.
9. Migrate export to Storage Access Framework.
10. Raise target SDK after export works without legacy storage.
