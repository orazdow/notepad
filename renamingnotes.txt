Renaming status
===============

The clone has been renamed from the original package:

    com.onto.notepad

to:

    com.clemm.notepad

This is sufficient for Android to treat it as a separate app from the installed
original, as long as the eventual Gradle project uses:

    namespace "com.clemm.notepad"
    applicationId "com.clemm.notepad"

Current source package
----------------------

All Java files now declare:

    package com.clemm.notepad;

and live under:

    app/src/main/java/com/clemm/notepad/

Current manifest registration
-----------------------------

The manifest package is:

    package="com.clemm.notepad"

Registered activities:

    com.clemm.notepad.NotepadNotesActivity
    com.clemm.notepad.NoteActivity
    com.clemm.notepad.SettingsActivity

The launcher activity has:

    android:exported="true"

Removed stale original-app wiring
---------------------------------

The old `NotepadApplication` manifest reference was removed because that class
only existed to initialize old billing/donation support and was not copied into
the clone.

The old Google Play billing permission, donate menu item, and donation/billing
activity wiring were also removed.

Still worth deciding later
--------------------------

The SQLite database is still named:

    NotepadDB

This is fine for side-by-side install because Android private app data is scoped
by application ID. Keeping the name also helps if you later import old data.

The export folder is still:

    /Notepad

That is shared external storage. Change it to `/NotepadClemm` or use Android's
file picker if you want exported text files to be clearly separate too.
